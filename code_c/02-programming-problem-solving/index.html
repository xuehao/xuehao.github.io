Chapter 2. 关于程序设计和问题求解

发表于 2018-10-24

在深入学习C语言编程细节之前，我们需要对程序设计和问题求解有个直观的认识。这样才能达到事半功倍的效果。

程序设计包含了很多抽象的概念，从长远来看，学习这些通用的方法更有意义；问题求解的过程使得程序设计有了更多趣味，设计求解策略本身便充满了很多挑战。有了这些求解策略，再转换成可运行的C程序便是水到渠成的事情。

所以，尽量保持一个整体的观念，从整体上把握程序的作用而不要拘泥于细节。在问题求解的过程中，慢慢丰富C语言的细节知识，从而精通C语言。
